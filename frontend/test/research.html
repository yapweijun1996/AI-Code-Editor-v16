Perform Research Test (Guide)

Run the multi-stage research flow from inside the main editor app UI.
← Back to Test Listing
1) Open the Editor App
Use the button below to open the main app in a new tab.
Open Editor App
Ensure the backend server is running and the app loads at http://localhost:3333/
2) Open AI Chat and Run Perform Research
In the appâ€™s AI Chat panel, send a prompt like:
perform_research: - query: "What are the main architectural patterns in software engineering?" - max_results: 3 - depth: 2 - relevance_threshold: 0.7
The app exposes the perform_research tool via the chat interface. The agent will execute searches, read URLs, and synthesize results, updating logs in the UI.
Optional: Run via Browser Console
If you prefer, in the app tab press F12 and run this snippet on the Console to invoke the tool directly:
import("/js/tool_executor.js").then(async m => { const toolCall = { name: "perform_research", args: { query: "What are the main architectural patterns in software engineering?", max_results: 3, depth: 2, relevance_threshold: 0.7 }}; const result = await m.execute(toolCall, null, false); console.log(result); });
This uses the in-app tool executor exactly like the chat would.
Troubleshooting
  • If searches fail, verify DuckDuckGo test is passing on the DuckDuckGo page.
  • Network or CORS issues can cause intermittent failures; try again or change the query.
  • If the UI does not update, reload the app tab and retry the snippet or chat command.
